Zaključak

Za ~2 dana rada isporučili smo kompletan redizajn Persona/Instructions taba unutar CustomGPT.ai agent editora. Krenuli smo od neresponsivnog, desktop-only layouta sa praznom textarea-om koja zbunjuje korisnike — i stigli do čistog, mobile-first interfejsa sa AI-driven chat modom, realtime sync-om, i UX-om koji jasno odvaja "šta agent radi" (Settings) od "kako agent priča" (Instructions).

1 Polazna tačka — Originalni dizajn

Na polaznom screenshotu vidimo šta je postojalo u produkciji:

Pre Problem
Jedna textarea za sistem prompt Korisnik mora sam znati šta je dobar sistem prompt — blank page paralysis
"Set Up Instructions" header Pasivan naslov, ništa ne objašnjava niti vodi korisnika
Persona Playbook + Best Practices kartice Vode korisnika van stranice na eksternu dokumentaciju
Cancel / Delete / Save Changes dugmad Standardni CRUD — nema preview feedback-a, nema znaka da li je promjena primenjena
Desktop-only layout Na mobilnom i tabletu layout se lomi
Nema AI pomoći Svaku instrukciju korisnik piše ručno, od nule

Suštinski problem: Samo napredni korisnici znaju kako da napišu efikasan sistem prompt. Svi ostali pišu nešto površno ili ne pišu ništa — agent ostaje nekonfigurisan.

2 Proces — 9 koraka transformacije

Korak 1 — Responsive layout (mobile-first)

Pomeren breakpoint sa lg (1024px) na xl (1280px) kao jedinstven prelaz između jednokolonskog i dvokolonskog layouta. Sidebar je pretvoren u fixed drawer sa overlay backdrop-om na mobilnom — na desktopu uvek vidljiv. Dodat bottom switcher (Settings / Preview) za mobilne korisnike.

Korak 2 — Uklanjanje Setup Guide-a

Uklonjen Setup Guide, Persona Playbook kartice i Best Practices kartice sa Settings taba. Sve što je vodilo korisnika van editora je izbačeno. Ostalo je 5 čistih sekcija: Identity, Personality, Behavior Rules, Output & Limits, Workflow Outcomes.

Korak 3 — PXO analiza → 3 buga identifikovana i fiksirana

Bug 1

Silent sync failure

Kad bi API call pao, status bi se tiho resetovao na idle. Korisnik ne bi znao da su promene izgubljene.

→ Error state + Retry dugme
Bug 2

Breakpoint bug

Layout se prelomio na lg (1024px) — na laptop ekranima 1100–1280px bio je broken.

→ Pomeren na xl (1280px)
Bug 3

Publish bez zaštite

Agent se mogao objaviti sa 0/5 completion score-om bez ikakvog upozorenja.

→ Two-step confirmation

Korak 4 — Instructions mod (bivši "Natural" → "Persona Studio" → "Instructions")

Najvažnija promena. Dodat je AI chat mod pored Settings taba. Korisnik opisuje u slobodnom jeziku šta želi, AI parsira zahtev i aplicira strukturirani patch na PersonaState.

Kritičan tehnički problem — feedback loop: generateInstruction() je uključivao additionalInstructions u svom outputu. BuilderChat je pisao u to polje. Svaki novi prompt sadržao je prethodni output → beskonačan rast.

Fix: Uklonjen additionalInstructions iz generateInstruction() potpuno. U PersonaEditor useEffect-u, base instrukcija i detailed instrukcije se spajaju tek na kraju, neposredno pre slanja API-ju.

Korak 5 — Suggestions UX redesign

Originalni dizajn: flat lista chipova bez strukture. Novi dizajn: uvek vidljiva kartica iznad inputa sa category tabovima (All / Support / Expert / Ops / Coaching...), expand modom, i Undo dugmetom posle svake primenjene promene.

Korak 6 — Razdvajanje Regular vs. Follow-up sugestija

Regular suggestions = uvek vidljiva kartica iznad inputa, koristi INITIAL_SUGGESTIONS. Follow-up suggestions = pojavljuju se inline u chat thread-u, ispod poslednje poruke, nestaju čim korisnik pošalje novu poruku.

Korak 7 — Diff tagovi kao read-only labele

Prikaz AI promena je originalno koristio chip dizajn koji je izgledao klikabilno. Korisnici su pokušavali da kliknu na izmene misleći da mogu da ih edituju. Fix: redesign u read-only kolapsibilne labele — "N changes ∨" tekst otvara listu sa ✓ dodata vrednost / ~~uklonjena vrednost~~ bez border-a ili background-a koji sugeriše interaktivnost. Kolapsiran prikaz je default stanje.

Korak 8 — Očuvano stanje chata pri prelasku tabova

Kritičan UX bug: svaki prelaz Instructions → Settings → Instructions brisao je celu konverzaciju.

Uzrok: Ternary rendering {condition ? <BuilderChat/> : <Settings/>} unmountuje komponentu pri svakom switchu — React briše celo interno stanje.

Fix: CSS show/hide umesto conditional render. Oba panela su uvek mountovana u DOM-u, samo se menja flex / hidden klasa. Nula gubitka stanja.

// Pre — brisalo stanje pri svakom switchu
{chatMode === "builder" ? <BuilderChat/> : <Settings/>}

// Posle — oba uvek mountovana, CSS kontroliše vidljivost
<div className={chatMode === "builder" ? "flex" : "hidden"}>
  <BuilderChat/>
</div>
<div className={chatMode === "user" ? "block" : "hidden"}>
  <Settings/>
</div>

Korak 9 — Suggestions card alignment prema input polju

Kada se kartica otvori, bila je šira od input polja — vizuelno neusklađena. Fix: CSS grid sa 2 kolone koji obuhvata i karticu i input. Kolona 1 = 1fr (input), kolona 2 = auto (Send dugme). Kartica živi u koloni 1 — identična širina kao input, bez JavaScript-a.

3 Arhitektura — Ključne odluke

Odluka Alternativa Zašto ovako
CSS show/hide za tab switch Lift state u parent komponentu Nula promene API-ja, BuilderChat ostaje enkapsuliran
additionalInstructions van generateInstruction() Flag za loop detection Čistije, nema edge case-ova, feedback loop je strukturno nemoguć
xl breakpoint (1280px) Prilagodba po md/lg Jedinstven prelaz, nema broken intermediate stanja
2-col CSS grid za suggestions+input useRef width sync (JS) Čisto CSS rešenje, responsive bez overhead-a
lastInstructionRef za retry Re-generisanje iz state-a Garantuje identičan payload pri retry-u, nema race condition-a

4 Šta je isporučeno

  • Potpuno responsivan layout (mobile → desktop)
  • AI chat za pisanje instrukcija (Instructions mod)
  • Realtime sync sa error state i Retry UI
  • Two-step publish zaštita (0/5 guard)
  • Suggestions kartica sa tabovima i expand modom
  • Follow-up sugestije inline u chat thread-u
  • Diff tagovi kao read-only kolapsibilni labeli
  • Očuvano stanje chata pri prelasku tabova
  • Suggestions kartica poravnata sa input poljem
  • Uklonjen Setup Guide i mrtvi sadržaj
